# ThumbnailMaker — AI-Powered YouTube Thumbnail Studio

An AI-powered full-stack web app that turns a **headshot + a short video idea** into ready-to-publish YouTube thumbnails — with real-time progress streaming.

> Upload your photo, type *"React JS full course in 5 hours"*, get 3 styled thumbnails in ~15 seconds.

---

## Features

- **AI background removal (rembg / U2Net)** — your headshot is auto-cut out and rendered as a clean silhouette in front of every design, no green-screen required.
- **Headshot-preserving compositing** — your real face is used directly (no AI hallucinated faces).
- **Gemini-powered design metadata** — titles, subtitles, color palettes, and layouts generated by Google Gemini.
- **Local Pillow compositor** — fast, deterministic 1280×720 output with vector gradients, silhouette-shaped accent glows, drop shadows, and topic-aware badges.
- **Real-time progress** — Server-Sent Events stream live job status to the UI.
- **Async background jobs** — FastAPI BackgroundTasks + SQLite for job tracking.
- **Persistent history** — every generation is saved and browsable.
- **ImageKit-hosted output** — every thumbnail is uploaded and shareable via CDN URL.
- **Clean modern UI** — React 19 + Vite + Tailwind v4, dark glassmorphism design.

## Getting Started

### 1. Clone

```bash
git clone https://github.com/<your-username>/ThumbNailMaker.git
cd ThumbNailMaker
```

### 2. Backend setup

```bash
cd backend
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS / Linux

pip install -r requirements.txt
copy .env.example .env         # Windows
# cp .env.example .env         # macOS / Linux
```

Fill in `.env`:

```env
IMAGEKIT_PUBLIC_KEY=...
IMAGEKIT_PRIVATE_KEY=...
IMAGEKIT_URL_ENDPOINT=...
GEMINI_API_KEY=...
UNSPLASH_ACCESS_KEY=        # optional
```

Run the backend:

```bash
uvicorn main:app --reload
```

Backend runs on `http://localhost:8000`.

### 3. Frontend setup

```bash
cd frontend
npm install
copy .env.example .env         # Windows
# cp .env.example .env         # macOS / Linux
npm run dev
```

Frontend runs on `http://localhost:5173`.

## API Endpoints

| Method | Path                           | Description                          |
|--------|--------------------------------|--------------------------------------|
| POST   | `/api/jobs`                    | Create a job (multipart form)        |
| GET    | `/api/jobs`                    | List all jobs                        |
| GET    | `/api/jobs/{id}`               | Get a single job                     |
| GET    | `/api/jobs/{id}/thumbnails`    | Get a job's thumbnails               |
| GET    | `/api/jobs/{id}/stream`        | SSE stream of job progress           |

### Create job — request

```
POST /api/jobs
Content-Type: multipart/form-data

prompt:        "React JS full course in 5 hours"
style_count:   2
headshot:      <image file>
```

### SSE events

```
event: job_started      data: {"job_id": 1}
event: step_progress    data: {"step": "Generating designs..."}
event: job_completed    data: {"job_id": 1, "thumbnails": [...]}
event: job_failed       data: {"job_id": 1, "error": "..."}
```

## How It Works

1. User uploads headshot + prompt + style count.
2. Backend creates a `Job` row, schedules a `BackgroundTask`.
3. **Gemini** generates `style_count` design objects (title, subtitle, colors, layout).
4. The **Pillow compositor** builds a 1280×720 JPEG per design:
   - Vectorized gradient background with accent glow blobs and vignette.
   - Headshot is cropped to a portrait aspect, given a glow + drop shadow + accent border.
   - Eyebrow badge ("5 HOURS • FULL COURSE"), main title, subtitle pill, topic chip — all derived from the prompt and Gemini metadata.
5. Each composed JPEG is uploaded to **ImageKit**.
6. The DB is updated and an SSE event is pushed to the frontend.
7. Frontend swaps the loading state for the results gallery.
